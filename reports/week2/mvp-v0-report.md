# MVP v0 Report

## Purpose

The purpose of MVP v0 was to establish the initial technical foundation for the project and verify that the core infrastructure works correctly.

The MVP focuses on:

* Telegram Mini App integration;
* frontend deployment availability;
* backend API foundation;
* environment validation;
* smoke-check execution.

At this stage, the application serves as a technical prototype rather than a complete product.

---

## Current MVP Features

The current MVP v0 includes:

* Telegram Mini App initialization;
* Telegram SDK connection;
* environment validation;
* frontend deployment accessible via URL;
* backend authentication API foundation;
* Swagger API documentation;
* smoke-check validation.

* Swagger API documentation.

## Relationship to MVP v1

MVP v0 serves as a technical foundation for MVP v1.

The current version validates:

* Telegram Mini App compatibility;
* deployment pipeline;
* frontend-backend integration preparation;
* authentication foundation;
* environment setup.

Future MVP iterations will include:

* full questionnaire functionality;
* fragrance recommendation logic;
* improved UI/UX;
* permanent backend deployment;
* expanded API functionality.

---

## Current Limitations

The current MVP has several limitations:

* backend deployment is temporary;
* questionnaire functionality is incomplete;
* recommendation logic is not fully implemented;
* no permanent hosting infrastructure yet;
* limited frontend functionality.

These limitations are expected for the MVP v0 stage.
