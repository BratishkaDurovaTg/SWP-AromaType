# Sprint Retrospective

## What went well

* We successfully demonstrated the current MVP to the customer and received detailed feedback during the Sprint Review and User Acceptance Testing.
* We improved the project based on the customer's comments from the previous Sprint, especially the questionnaire, recommendation flow, and overall user experience.
* Documentation was kept up to date, including the roadmap, Definition of Done, customer review reports, and Sprint documentation.

## What did not go well

* Some features still required further refinement after the Sprint Review based on customer feedback, especially the questionnaire, checkout flow, and administrator functionality.
* Several UI issues were identified during User Acceptance Testing and needed additional improvements before the next Sprint.
* Some planned functionality had to be postponed because higher-priority improvements were identified during the customer review.

## What changed compared to the previous Sprint

Compared to the previous Sprint, we focused more on validating the product with the customer through a structured Sprint Review and User Acceptance Testing. We improved the questionnaire, recommendation flow, and user interface based on earlier feedback, and updated the Product Backlog with the new requests and priorities discussed during the review.

## Process improvements for the next Sprint

* Validate new features with the customer earlier so feedback can be incorporated before the end of the Sprint.
* Continue refining the Product Backlog throughout the Sprint to keep priorities clear and make Sprint Planning more efficient.
