# Customer Meeting Summary

**Date:** 26 June 2026

**Participants:**

* Team 6
* Customer

## Sprint Goal Reviewed

The Sprint Goal was to deliver a functional prototype of the Telegram Mini App that demonstrates the updated questionnaire, recommendation algorithm, and ordering workflow for customer evaluation during the Sprint Review.

## Delivered Product Increment

The development team demonstrated:

* Updated questionnaire based on the agreed reference article.
* Improved user interface.
* Recommendation algorithm based on four personality types.
* Telegram Mini App prototype.
* Initial recommendation page.
* Prototype ordering workflow.
* Initial administrator catalog management concept.

## UAT Results

The customer completed User Acceptance Testing during the meeting.

Results:

* The application functioned correctly during the demonstration.
* The updated design was positively received.
* The questionnaire was considered significantly better than the previous version.
* The recommendation workflow operated as expected.
* Several usability improvements were identified for future implementation.

## Quality Evidence Discussed

The following quality evidence was reviewed:

* Successful demonstration of the working prototype.
* Functional questionnaire execution.
* Recommendation algorithm explanation.
* Customer-executed UAT.
* Identification of a frontend display bug (incorrect percentage values).
* Review of the current user flow and ordering process.
* Discussion of quality improvements planned for future CI and automated testing.

## Customer Feedback

The customer provided the following feedback:

* Some questionnaire answers reveal recommendation categories too directly.
* Answer options should be reordered to reduce obvious personality mapping.
* Product catalog should include real products, images, and richer descriptions.
* Unnecessary interface elements should be removed.
* Administrator interface should prioritize functionality over visual appearance.
* Ordering should follow a standard e-commerce checkout flow.
* Payment integration may be implemented in a future iteration.

## Approvals and Requested Changes

### Approved

* Updated UI design.
* New questionnaire concept.
* Recommendation methodology.
* Overall project direction.
* Manual assignment of personality categories within the administrator panel.

### Requested Changes

* Remix questionnaire answer ordering.
* Fix frontend percentage display.
* Remove redundant interface elements.
* Add real catalog data.
* Implement administrator catalog management.
* Simplify delivery by collecting only the customer's city.
* Prepare checkout for future payment integration.

## Remaining Risks

* Product database is not yet available.
* Checkout workflow requires further refinement.
* Payment integration remains outside the current Sprint scope.
* Delivery logistics require additional design in future Sprints.
* Product catalog still requires real content.

## Action Points

* Update answer ordering.
* Fix frontend UI issues.
* Implement administrator catalog management.
* Populate the catalog with real products.
* Simplify checkout information collection.
* Continue preparation for payment integration.
* Continue customer validation during future Sprint Reviews.

## Meeting Outcome

The customer accepted the demonstrated Sprint increment, approved the overall project direction, and confirmed the requested Product Backlog updates for the next Sprint.
