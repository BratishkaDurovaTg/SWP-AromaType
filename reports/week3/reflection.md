# Week 3 Reflection

## Learning Points

This week, we learned how to move our user stories from a document into GitHub Issues and organize them in a Product Backlog. We also learned how to write acceptance criteria, create technical PBIs, and use GitHub Projects to track work.

Working on the backlog helped us better understand how Scrum artifacts are connected and how important it is to keep requirements, issues, and documentation synchronized.

## Validated Assumptions

We confirmed that the questionnaire should remain the main way users receive fragrance recommendations. Customer feedback also supported our decision to focus on core functionality before adding additional features.

We also confirmed that backend deployment, database setup, and frontend-backend integration are necessary priorities before implementing more advanced functionality.

## Friction and Gaps

One difficulty was coordinating planning activities and keeping all backlog information updated. Some requirements still need clarification, and a few technical decisions will need further discussion during implementation.

The integration between frontend and backend is still a major area of work, and some backlog estimates are still being finalized.

## Planned Response

In the next Sprint, we plan to continue implementing MVP v1 features, complete backlog estimation, and improve integration between system components.

We will also continue refining the backlog and updating project documentation as new information and customer feedback become available.
